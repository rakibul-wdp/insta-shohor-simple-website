Javascript work by v8 engine. Which is basically google chrome's own. It's a client-side scripting language. It means that it runs on the client machine inside the web-browsers. when the browser loads a web page, parses the HTML and creates Document Object Model (DOM). And then the browser will grab linked files with HTML, like CSS files. It's come from a CSS parser. DOM creates a web page putting HTML CSS together. Then browsers load JavaScript files.

After done this work, JavaScript is executed in order the code is written. As a result, the DOM is updated through JavaScript and rendered by the browser. And also JavaScript reads code line by line. It's called synchronous. But remember that JS also runs code asynchronously. It is a single-thread programming language, meaning it has a single stack of calls. Therefore, it can do one thing at a time.

The event loop is the secret behind JavaScriptâ€™s asynchronous programming. JavaScript event loop is a system that says which process JavaScript code run. It collects processes and executes code. If you think of a scenario like a heap, engine, web API, callback queue. First of all JavaScript code runs synchronously. But it's not true all the time. In some cases, JavaScript code runs asynchronous (fetch, timeOut, etc). When the engine sees an asynchronous function or method it sends a web API for waiting.

After finishing synchronously code web API send callback queue and lastly render queue (event loop) send the code in the engine. Finally, it run. There are a few kinds of worlds related to event loop like stack(function calls form a stack of frames), Heap(large region of memory), Queue(list of code to be processed), Run to completion. And also in this process no time waste user.
